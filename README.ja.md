# HeptaSieve

**プライバシー優先、AI セーフ。Heptabase を構造化 Markdown へ継続同期するローカルツール。**

AI エージェントがどのカードを見られるかを、あなたが正確に決めます。それ以外は一切エージェントの視界に入りません。

[English](README.md) · [繁體中文](README.zh-TW.md) · [简体中文](README.zh-CN.md) · 日本語

> 非公式ツールです。Heptabase との提携・承認関係はありません。現時点では macOS のみ対応。

---

## なぜ作ったか

きっかけは単純でした。Heptabase を Claude Code につなぎ、AI エージェントに自分のノートを読ませたかったのです。

公式の経路（Heptabase MCP サーバーやコミュニティ製 CLI）はいずれも **fail-open** です。一度認可すると、エージェントはナレッジベース全体を読めてしまいます。すべてが共有して問題ないなら構いませんが、多くの人がそうであるように、AI に使わせたいカードのすぐ隣に機密カードを置いている場合、この経路は成り立ちません。

本質的な気づきはこうです。プライバシーの壁は Heptabase の内部にあるのではなく、「AI が何を読めるか」という境界にあります。だからこの種のツールの価値は「ノートを同期する」ことではなく、**機密カードを AI が届かない場所に置いたまま、残りだけを AI が読める Markdown に書き出す** ことです。

これが sieve（ふるい）です。あなたが通したカードだけが通り抜けます。

## できること

HeptaSieve はローカルの Heptabase データベースを直接読み、選んだカードを Markdown ファイルとして任意の場所に書き出します。`launchd` が 15 分ごとに実行し、Markdown をノートと同期し続けます。AI エージェントは書き出された Markdown フォルダだけを読み、データベースには一切触れません。

- **ライブのローカル DB を直読み。** Heptabase は 2025 年末にローカルバックアップの提供を終了したため、ライブ DB の直読みが継続的なローカル同期の確実な手段になりました。
- **構造を保った変換。** 表、bullet／todo／toggle リスト、ネストした section、動画を、Heptabase の ProseMirror スキーマからリバースエンジニアリングし、きれいな Markdown へ変換します。
- **任意の宛先へのルーティング。** 各ホワイトボードを個別のフォルダへ落とせます。絶対パスを使えば、特定のボードを別プロジェクトへ直接配置することも可能です。

## fail-closed プライバシーモデル

カードは、次の 2 つの明示的な許可リストのいずれかに一致した場合のみ書き出されます。デフォルトでは何も読みません。

| ソース | ルール |
|---|---|
| **`whitelist_whiteboards`** | 指名したホワイトボード。各ボードの『表面』にあるカードだけを読み、サブホワイトボードは辿りません。読みたいサブボードはその名前も追加します。 |
| **`card_map`** | `タイトル -> 正確なパス` の上書き層。これらのタイトルは常に同期され、パスが優先されます。 |
| **`blacklist_whiteboards`** | これらのボード上のカードは、内容を読む『前に』差し引かれます。ブラックリストはホワイトリストに優先するため、2 つのボードに誤って置いたカードも遮断されます。 |
| **指名していないサブホワイトボード** | カードをサブボードへ移すと `whiteboard_id` が変わるため、表面スキャンでは見えません。ルールを覚える必要のない、構造による除外です。 |

一言での保証：カードのタイトルや内容に触れるクエリはすべて、ホワイトリストのボード id か `card_map` のタイトルに限定されます。非ホワイトリストのカードのタイトルと内容は、そもそもメモリに読み込まれません。

ここから 2 つの設計原則が導かれます。**構造的除外は減算的除外に勝る**（クエリが届かないカードは、読み込んでから除外するより安全）、そして **最良の通知は不要な通知**（「あのカードを漏らさなかったか？」と問う必要が生じないように設計する）。

## 比較

| | HeptaSieve | 公式 Heptabase MCP / CLI | その他のエクスポートツール |
|---|---|---|---|
| プライバシーモデル | fail-closed 許可リスト | fail-open（ナレッジベース全体） | 全量エクスポート |
| 継続的ローカル同期 | あり（`launchd`、15 分） | 要求時に読み取り | 一回限り |
| ライブのローカル DB を直読み | あり | 場合による | バックアップファイルが必要なことが多い |
| 構造を保った Markdown | 表・リスト・section・動画 | 場合による | 場合による |
| ボードごとの宛先ルーティング | あり、絶対パス対応 | なし | なし |

これは Heptabase の完全な代替ではありません。「公式より良い」と言えるのは 3 つの軸だけです。制御可能なプライバシー、継続的なローカル同期、構造の忠実さ。対象はあえて狭く、AI に何を見せるかを気にする Heptabase ヘビーユーザーの macOS 利用者です。それがあなたなら、このツールはまさにあなたの状況のために作られています。

## インストール

要件：macOS、Python 3.9+、Heptabase デスクトップアプリのインストール。

```bash
git clone https://github.com/yyu0310/heptasieve.git
cd heptasieve
cp config.example.json config.json
```

次に `config.json` を編集します（各フィールドにはインラインの説明コメントがあります）。

1. `db_path` がローカルの `hepta.db` を指しているか確認する。
2. `base_output_dir` と `board_output_dir` を Markdown の書き出し先に設定する。
3. 書き出したいホワイトボードを `whitelist_whiteboards` に列挙する。
4. 正確なパスで上書きしたいタイトルを `card_map` に追加する。

まずは何も書き込まないプレビューを実行します。

```bash
python3 heptabase_sync.py --dry
```

計画が正しければ、本番実行します。

```bash
python3 heptabase_sync.py
```

### 15 分ごとに自動同期

```bash
cp com.example.heptasieve.plist ~/Library/LaunchAgents/
# コピーしたファイルを編集：絶対パスを設定し、python3 のパスを確認する
launchctl load ~/Library/LaunchAgents/com.example.heptasieve.plist
```

## AI エージェントと一緒に使う

HeptaSieve はエージェントが読めるドキュメントを同梱しています。手順を手で追う代わりに、AI コーディングエージェントとの対話でセットアップできます。

- [`AGENTS.md`](AGENTS.md) と [`CLAUDE.md`](CLAUDE.md)：エージェントがこのツールをどう理解し設定すべきか。
- [`llms.txt`](llms.txt)：LLM 向けのドキュメント索引。
- [`skills/setup-heptasieve/`](skills/setup-heptasieve/)：一度の依頼でセットアップ全体を案内する Claude Code スキル。

エージェントは書き出された Markdown フォルダに向け、`hepta.db` には決して向けないでください。この分離こそが、このツールの肝です。

## 仕組み

アーキテクチャの詳細は [`ARCHITECTURE.md`](ARCHITECTURE.md) を参照してください。データフロー、`build_plan` 内の fail-closed の順序、読み取るデータベーステーブル、コード変更時に守るべきプライバシー不変条件を説明しています。

## 制限と正直な注意点

- **スキーマは壊れやすい。** Heptabase 内部のデータベース構造に依存します。Heptabase の更新で動かなくなる可能性があります。本質的に非公式です。
- **ライブ DB の直読みは公式に認められていません。** 実用上はよく動き読み取り専用ですが、サポートされた連携方式ではない点は理解しておいてください。
- **macOS のみ。** 現在のパスと `launchd` の設定は macOS を前提にしています。
